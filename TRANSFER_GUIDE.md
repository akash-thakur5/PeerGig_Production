# PeerGig Project Transfer Guide

To share the complete PeerGig project (code + database data) with your friend, follow these steps.

## 1. Prepare the Files

The project contains everything needed to run on another machine. You should share the following:
- **Project Folder**: Zip the entire `PeerGig` folder.
- **Exclusion List** (Do NOT include these in the zip):
  - `node_modules/`
  - `.next/`
  - `.git/`

## 2. Include the Database Backup
I have created a full database dump of your current data:
- **Location**: `db/backup.sql`
- This file contains all your users, gigs, wallet balances, and study notes.

## 3. Include Environment Variables
- Ensure you share the `.env.local` file with them, as it contains necessary secrets and connection strings.

## Instructions for your Friend (on the new laptop)

Once your friend has the files, they should:

### 1. Setup the Environment
- Ensure **Node.js** and **Docker Desktop** are installed.
- Open the project folder and run:
  ```bash
  npm install
  ```

### 2. Start the Database
- Run the following to start the database container:
  ```bash
  docker-compose up -d
  ```

### 3. Restore the Data (CRITICAL)
- To load all the shared data into their database, run:
  ```bash
  docker exec -i peergig-db psql -U peergig -d peergig < db/backup.sql
  ```

### 4. Run the Website
- Start the development server:
  ```bash
  npm run dev
  ```
- Visit `http://localhost:3000`.
